18.14. Error Handling

exit_on_error (boolean)

如果为真，那么任何错误都将终止当前事务。缺省时，设置为假，所以只有致命错误将终止会话。

restart_after_crash (boolean)

当缺省设置为真时，PostgreSQL将在后端崩溃后自动重新初始化。 设置这些值为真往往是优化数据库的最佳方式。然而，在某些情况下，比如当PostgreSQL 正由集群调用时，它可能禁用启动非常有益，使得集群可以得到控制并采取其认为适当的任何行动。
