Severity 类是打印属性类，它是一种枚举，表示 PrinterStateReason 属性的严重性。

Severity 的实例不会直接出现在 Print Service 的属性集中。而 PrinterStateReasons 属性则会出现在 Print Service 的属性集中。PrinterStateReasons 属性包括零、一个或多个与 Print Service 的状态相关的 PrinterStateReason 对象，并且每个 PrinterStateReason 对象都与 REPORT（不太严重）、WARNING 或 ERROR（最严重）的 Severity 级别相关。当相应的条件变成了打印机的真实条件时，打印机将把 PrinterStateReason 对象添加到 Print Service 的 PrinterStateReasons 属性中，而当相应的条件变成虚假条件时，无论 Print Service 的总体 PrinterState 是否也已更改，打印机将再次移除 PrinterStateReason 对象。

IPP 兼容性：Severity.toString() 返回 "error"、"warning" 或 "report"。字符串值由每个 PrinterStateReason 和相关的 Severity 对象的 toString() 方法返回，它们之间用连字号（"-"）连接，该值提供了 PrinterStateReasons 的 IPP 关键字值。由 getName() 返回的类别名称提供了 IPP 属性名称。

字段详细信息

REPORT

public static final Severity REPORT
指示 PrinterStateReason 是 "report"（不太严重）。实现可以选择忽略部分或全部 "report"。有些 "report" 指定了更为详细的打印机状态；其他一些可作为 "warning" 的先兆。"report" 不能包含任何会影响打印输出的内容。

WARNING

public static final Severity WARNING
指示 PrinterStateReason 是 "warning"。实现可以选择忽略部分或全部 "warning"。"warning" 可作为 "error" 的先兆。"error" 不能包括任何会阻止作业完成的内容，但在某些情况下输出的质量可能仍然较差。

ERROR

public static final Severity ERROR
指示 PrinterStateReason 是 "error"（最严重）。实现必须包括全部的 "error"。如果此属性包括一个或多个 "error"，则打印机的 PrinterState 必须为 STOPPED。
